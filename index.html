    一、請問var、let、const的差別為何？能舉出例子嗎？

    答案：

    var可以在全域或函式作用域、區塊作用域(循環語句)內使用，而var若在函式內宣告了一個變數名稱時，實際上是會下在window物件上，變成最高級的window物件的屬性， 因此後續將不可變更，例如在函式內宣告了一次var a = 1 ，在函式外打印a則只會提示a is not defined。此意即"var的全域污染性"。

    另外，var所宣告的變數名稱，可以再次透過var宣告該相同變數名稱。

    var在JS中存在"提升"的用法，意即var a = 1的情況下，創造階段的var a會被提升到作用域頂部，透過預留的記憶體空間先給予a其undefined的值。

    let宣告了一個變數名稱之後，在同一個作用域底下，不能再次使用let宣告同一個變數名稱，但是可以使用"變數名稱=某值"的方式修改。

    而在不同的作用域中，則可以被重複使用let宣告，例如全域作用域和函式作用域，即可重複使用let a = 1。

    const適合用在不應修改的變數上，跟let一樣在同一個作用域底下，不能再次使用const宣告同一個變數名稱，而且不可以使用"變數名稱=某值"的方式修改。

    const宣告之變數則要求必須有值否則會報錯 但特例是因為物件是"傳參考"特性，若const宣告某變數名稱=某物件，則還是能夠修改其物件屬性，若要禁止修改則必須使用Object.freeze(變數名稱)的方式去凍結物件內容。

    二、請使用map、forEach實作出[1,2,3]迴圈。

    答案：

    map() 方法會建立一個新的陣列，其內容為原陣列的每一個元素經由回呼函式運算後所回傳的結果之集合。

    const array = [1, 2, 3];

    const map = array.map(x);

    console.log(map); 會印出Aarray = [1, 2, 3];

    forEach() 方法會將陣列內的每個元素，皆傳入並執行給定的函式一次。

    const array = [1, 2, 3];

    array.forEach(element => console.log(element));

    會印出
    1
    2
    3

    【問答題】

    一、請解釋一下何謂同步與非同步？

    答案：

    JS為單執行緒，事件會依照順序排入事件佇列(queue)，等待瀏覽器做完一件事之後才能做下一件事，就算中間卡了一個迴圈也一樣得等，此即同步。 而諸如瀏覽器所提供的功能，像是setTimeout( )這種有設定timer功能的函式，因為需要等待倒數計時，因此會例外被放到最後才加入事件佇列，即便它設定0秒也一樣，此即非同步。

    二、你對ES6瞭解有多深？

    答案：

    1.新增了let、const的新宣告方法

    2.可以使用(、${變數名稱}、)的樣板字面值，方便字串拼接，也可以拿來寫函式、判斷式。

    3.可以透過let [序號]=arr;的方式取用arr中的值，也可以透過let {animal:{dog}}=obj;的方式來取出obj中的屬性，和其下的屬性。

    4.透過[...arr]，可以快速將arr陣列中的值展開，也可以透過[x,...z]或{x,...z}的方式取代剩餘的參數，意即其餘參數，但其餘參數必須放在最後。

    5.可以在宣告或使用函式時幫參數加上預設值。(例如lat {a='狗' ,b='貓' })=obj;

    6.新增了箭頭函式。 只有一個回傳值時，可以省略return，但如果有使用大括號 {} 包覆就還是必須加上 return，不然傳入的值會印出 undefined。 只有一個參數時，則可以省略括號 ( )。

    另外，箭頭函式內的this會等於外層函式的this。

    7.可以使用export加在想輸出的物件、類別、函式與原始資料(變數與常數)之前，並使用import加上{}部分引入或*+自定義的模組名稱進行全部引入。

    8.建議可以安裝Babel轉譯器協助轉成需要的ES版本。

    三、若Git Flow上，同事於前一晚已在Dev主分支上有commit的紀錄， 你隔天上班發現你開出feature旁分支並且修改內容commit後才發現此事， 但同事又希望維持一個主分支應如何處理？

    答案：

    1.若從Dev方使用merge合併feature後，仍會留下一個醜醜的feature分支， 此時應推薦用 git pull --rebase 的方式，才不會留下一個醜醜的feature分支， 而與merge不同的是，merge只要解衝突一次， 但rebase把你把本地的repo從上次pull之後的所有變更都先暫存起來，然後回復到上次pull時本地的repo狀態， 之後才會去套用遠端repo的變更，最後再套用你剛剛暫存下來的本地repo變更， 因此rebase過程中如發生衝突則會暫停rebase，需要你去修改解決你自己本地repo後來變更內容的衝突。

    2.但如果你變更的內容比較多，預期會有較多的衝突，建議還是用merge。
    如果你變更的內容比較少，不太預期有衝突，則建議用rebase，等於把你自己的commit砍掉重新commit。

    四、請簡述RESTful API用途是什麼？

    答案：

    RESTful API全稱為Representational state transfer Application Programming Interface， 意即表現層狀態轉換應用程式介面， RESTful 其實是指它的API設計風格。

    核心有三大訴求：

    1.資源位置的URL，每個資源都會有唯一的URL位置。

    2.對資源的取用動作，每個資源都會有GET、POST、PUT、DELETE等動作。

    3.資源呈現的格式：API資源可以以YAML、XML、JSON等方式呈現。

    RESTful API將瀏覽器作為client端口，統一請求的界面格式，使其具備可重用的優點， 並利用cache來做到存取速度優化，也讓界面與資料分離，並達到節省伺服器計算成本的好處。

    其用途在於若有多組資源需要被不同平台使用時， 為了統一標準，避免DB部分重工， 可以透過RESTful API的方式統一使用HTTP協定操作， 因此API方只要知道各平台的http-client， 平台方只要配合相應的資料格式，即可做到各平台支援。

    JS 核心篇內功心法

    2022年2月27日胖胖龍爹地


    《JS核心篇 內功心法》

    1.純值無法新增屬性，若非純值，即為物件(包含函式)。

    2.以變數賦值變數，為傳值；以物件賦值物件，為傳參考。

    3.為避免兩個物件間的傳參考特性，一律用深層拷貝 JSON.parse(JSON.parse())最保險。

    4.JSON的屬性和值其實都是雙引號包裹的字串，便於網路傳遞，但可透過框架轉為JS物件。

    5.具名函式的函式名稱無法在函式外被調用。

    6.函式若可被正常執行即毋須具名。

    7.立即函式無法在函式外執行。

    8.簡易呼叫的this盡可能不要調用，因為它的本質其實是undefined。

    9.var的作用域是函式的作用域，所宣告者會成為全域變數， let的作用域是大括號內的block作用域， 同一作用域下不可使用let重複宣告同一變數，也只活在大括號內。

    10.非同步者如setTimeout會優先放入事件緒列，暫不執行。

    11.以const宣告變數為物件，則物件形同被凍結，無法刪改屬性，也無法被改為新物件。

    12.JS賦值具有Hoisting特性，創造階段形同先賦予undefined的值，待到執行階段才給予實際的值。

    13.以let賦值雖有Hoisting特性，但於創造階段不會先賦予undefined的值，因此產生一個暫時性死區(TDZ)， 使你無法去存取此變數因此報錯，直到實際宣告值的階段才結束。

    14.若對於ES6語法不熟悉，建議團隊統一使用ESlint給予撰寫規範提示，規範原因可參閱ESlint文檔， 請透過Node.js於終端機安裝npm install eslint --global。

    15.let宣告變數後未賦值，取用時是提示undefined，而const在宣告時需要賦予值，所以取用時是提示語法錯誤。

    16.箭頭函式與傳統函式相比，箭頭函式沒有自己的this，也無法透過call、apply、bind重新給予this， 因此若箭頭函式內有this，則該this指向箭頭函式的外層，因此需改為傳統函式function才可正確指向。

    17.箭頭函式不具有prototype，因此也無法作為建構函式使用。

    18.箭頭函式無法回傳出物件實字。

    19.判斷式中無法直接搭配箭頭函式，箭頭函式需搭配中括號，以免結構錯誤，如Vue生命週期之使用應寫入避免箭頭函式。

    20.使用prototype函式新增屬性時，應避免使用箭頭函式，因為該this將指向全域。

    21.ES6樣板字面值是為了解決過去字串用+號組合閱讀不易的問題。

    22.樣板字面值：將字串以反引號、、(ctrl+')包覆，中間的變數則輸入${ 變數名稱 }。

    23.${ 變數名稱 }也可加入判斷式，例如${ 變數名稱 || 變數名稱 }。

    24.傳統寫法的ul、li標籤語尾須加上換行符號\才可換行，ES6語法使用反引號包覆ul、li標籤後則不需要。

    25.VSC左下settings，右上角open settings，進入settings.json，在 setting.json 內的物件內加入emmet設定檔之後，即可在樣板字面值內使用emmet產生HTML結構。

    26.${ 變數名稱 }中可以再插入${ 變數名稱 }，作為巢狀。

    27.若函式中的參數數量不確定，可加入...arg作為其餘參數，後續打印只要傳入arg即可回傳出陣列。

    28.promise用來處理非同步的行為，例如AJAX取得遠端資料即可搭配使用。

    寫法: 先宣告一個函式並return透過new建立的一個Promise物件，並使之帶有resolve和reject參數， 後續再呼叫其函式，此時promise在資料接收到之前為pending未確認狀態， 直到進入settled已確認狀態，會出現fulfilled的已實現狀態或rejected的已否決狀態， 在執行結果成功時，為fulfilled的已實現狀態，promise會透過resolve參數回傳結果並透過調用的函式以then接收， 而若是執行結果失敗，則會出現rejected的已否決狀態，promise會透過reject參數回傳結果並透過調用的函式以catch接收。

    29.axios是基於promise的遠端請求工具，但使用前須先載入axios套件。

    30.jQuery也有自己的AJAX方法，但使用前須先載入jQuery套件，而且寫法不只一種。

    31.非同步行為中如果再插入非同步行為，會形成巢狀的回呼地獄，因此改以axios的get方法，可以使第二次的回呼行為改以第二個then串接。

    32.非promise基礎的套件，會有寫法不一致的問題，因此改以axios的get方法，可以使所有promise一律用then串接。

    33.非promise基礎的套件，會有發出請求和結束時間不一致的問題，因此改以promise的all方法帶入axios.get(url)作為陣列， 後續以then串接帶入[res1, res2]，始之可以一同送出、一同回傳。

    34.Promise Chain的串接方法為在第一個then底下return函式的另一個值，之後在下方補上第二個then方法， 但若其中一個return出錯或失敗，則會直接跳到catch結果，catch後若有接return則可以執行新一個then。

    35.then其實可傳入兩個callback function， 因此可以寫入第一個then(res) => {} 作為成功接收結果， 並後頭加上第二個(rej) => {} 作為失敗接收結果。

    36.Callback函式 (Callback function)，CallBack又稱為回調、回函、回呼函式， 在某一函式執行時或執行後，依條件或依序去執行所傳遞進來的函式。就是一個程式執行完再去執行另一個。

    37.若需要同時發出多個api請求，可使用Promise.all([])，並傳入promiseFn(x, 毫秒數字)加入陣列， 回傳結果可以用then接收，並打印出各個(res[0], res[1], res[2])， 但只要有其一被拒絕則會進入catch方法。

    38.Promise.race([])，則只會回傳第一個完成結果，因此只要打印一個res即可， 但只要有"第一個"被拒絕的結果則會進入catch方法。

    39.XMLHttpRequest建構函式可產生物件作為遠端請求使用，但原始碼並不好管理，建議可改為promise寫法， 只要傳入url即可，並使用resole(req,response)、reject(req)以及get(url)加上then及catch方法來取得結果， get也可補上return get(url)並加上第二個then來串接使用。

    40.{JSON} Placeholder會產生假資料提供API學習使用。
